## <Title>🌱 들어가며</Title>

현업에서 개발을 진행하면서, 팀원들과 함께 시나리오 기반 단위 테스트 문서를 작성하고 직접 테스트해본 경험이 있습니다. 하지만 이렇게 수동으로 테스트하다 보니 시간이 오래 걸리고, 실수로 놓치는 부분도 종종 있었습니다.

그래서 이번에는 테스트를 자동화하는 방법에 대해 정리해보려고 합니다. 자동화 테스트를 활용하면 반복적인 확인 작업을 줄이면서도, 코드가 의도대로 동작하는지 안정적으로 검증할 수 있습니다. 이번 글에서는 React 프로젝트에서 Cypress를 활용해 실제 사용자처럼 앱을 테스트하는 방법을 소개하겠습니다.

## <Title>🌱 테스트 도구 선택하기</Title>

리액트 프로젝트에서 테스트를 작성할 때는 목적과 범위에 따라 다양한 도구를 선택할 수 있습니다. 대표적으로 많이 쓰이는 도구는 다음과 같습니다.

|             | Jest + React Testing Library                       | Cypress                                                                     |
| ----------- | -------------------------------------------------- | --------------------------------------------------------------------------- |
| 테스트 범위 | 단위(Unit) / 통합(Integration)                     | E2E(End-to-End)                                                             |
| 테스트 대상 | 컴포넌트 단위                                      | 전체 사용자 흐름, 페이지 단위                                               |
| 실행 환경   | Node.js 환경에서 DOM을 시뮬레이션                  | 실제 브라우저에서 실행                                                      |
| 기타        | CRA로 시작하면 기본적으로 포함됨<br/>빠르고 가벼움 | 브라우저 DevTools 사용 가능<br/>컴포넌트 테스트도 가능하지만 추가 설정 필요 |

👉 어떤 상황에 어떤 도구를 써야 할까?

- 컴포넌트 단위의 로직 테스트 → Jest + RTL
- 실제 사용자 흐름 검증 (로그인, 결제 등) → Cypress
- 배포 전 전체 시나리오 점검 → Cypress
- 사이드 프로젝트나 빠른 개발 → Jest + RTL로 충분

## <Title>⚙️ Cypress 사용해보기</Title>

### 1. Cypress 설치

```shell
npm install cypress --save-dev
```

### 2. Cypress 열기

```shell
npx cypress open
```
